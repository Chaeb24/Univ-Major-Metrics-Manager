# 🏢 대학 정보 및 학과별 공시 지표 통합 관리 시스템 
##(Univ-Major-Metrics-Manager)
### 미니 프로젝트

> **Java와 Oracle DB(JDBC)를 활용하여 대학, 학과 정보, 그리고 연도별 세부 지표(경쟁률·취업률)를 관리 & 분석할 수 있는프로그램**
> **기본 CRUD만 정의한 상태

---

## 📌 1. 프로젝트 개요
본 프로젝트는 대학 입시 및 학과 정보 분석의 기초가 되는 대형 데이터를 효율적으로 처리하기 위해 설계
단순한 데이터 조회를 넘어 RDB의 계층형 구조(1:N 관계)를 명확히 이해하고, 다중 테이블 JOIN 및 제약 조건을 활용하여 
데이터 무결성을 보장하는 백엔드 시스템을 구현하는 것을 목표로 합니다.

---

## 🛠 2. 사용 기술 스택 (Tech Stack)
* **Language:** Java (JDK 17)
* **Database:** Oracle DB
* **Library/Framework:** JDBC (Java Database Connectivity), Lombok
* **Architecture:** MVC Pattern 기반 아키텍처 (Controller - Service - DAO - DTO - View 구조 분리)

---

## 📐 3. 데이터베이스 설계 (ERD)
데이터 간의 유기적인 종속 관계를 반영하여 **대학(부모) ➔ 학과(자식) ➔ 지표(손자)** 형태로 테이블을 구조화했습니다.

* **University (대학):** 대학명, 주소, 연락처, 홈페이지 URL, 입학처 URL
* **UniversityMajor (학과):** 학과명, 지역, 주/야간 구분 (`universityIdFK` 외래키 참조)
* **UniversityMajor_Metrics (지표):** 공시연도, 경쟁률, 재학생수, 입학정원, 취업률 (`universityMajorFK` 외래키 참조)
* **데이터 무결성 방침:** 대학 데이터 삭제 시 `ON DELETE CASCADE` 제약 조건을 활용하여 하위 학과 및 지표 데이터까지 종속적으로 일괄 자동 삭제되도록 처리했습니다.

---

## 🔥 4. 핵심 기능 (Key Features)

### 🏢 대학 정보 관리 (CRUD)
* **등록/수정:** 대학명, 주소, 연락처 등을 입력받아 오라클 시퀀스를 통해 고유 ID를 부여하여 생성.
* **조회:** 전체 대학 요약 목록 조회 및 특정 대학명 검색을 통해 정보 조회 가능.
* **삭제:** 대학 고유 ID로 삭제를 수행하며, CASCADE 규칙에 의해 관련 학과/지표가 연쇄 삭제됩니다.

### 🎓 학과 정보 관리
* **대학별 개설 학과 필터링 조회:** 2개 테이블 JOIN을 통해 사용자가 특정 '대학 이름'을 입력하면 해당 대학에 개설된 모든 학과 목록만 출력.
* **소속 기반 등록:** 부모 대학 고유 ID(FK)를 매핑하여 새로운 학과를 등록합니다.

### 📈 연도별 학과 지표 분석 (정밀 조건 검색)
* **지표 전체 조회:** 학과별 연도 데이터(경쟁률, 취업률, 정원 등)를 표 형태로 일괄 출력합니다.
* **정밀 타겟 검색 (AND 조건):** 3개 테이블 대통합 JOIN 엔진을 기반으로, 사용자가 검색하고자 하는 **'대학명'과 '학과명'을 동시에 만족하는 정확한 단건의 성적표(지표)**를 찾아내어 가독성 있게 매핑 후 출력합니다.
